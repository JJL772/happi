Command Line Interface

usage: happi [-h] [--path PATH] [--verbose] [--version]
             {search,add,edit,load,update,container-registry,transfer} ...

Positional Arguments

cmd

Possible choices: search, add, edit, load, update, container-registry, transfer

Subcommands used to search, add, edit, or load entries

Named Arguments

--path

Provide the path to happi configuration file.

--verbose, -v

Show the debug logging stream.

--version, -V

Show the current version and location of Happi installation.

Sub-commands:

search

Search the happi database.

happi search [-h] [--json] [--names] search_criteria [search_criteria ...]

Positional Arguments

search_criteria

Search criteria of the form: field=value. If “field=” is omitted, it will be assumed to be “name”. You may include as many search criteria as you like; these will be combined swith ANDs.

Named Arguments

--json

Show results in JSON format.

--names

Return results as whitespace-separated names.

add

Add new entries interactively.

happi add [-h] [--clone CLONE]

Named Arguments

--clone

Copy the fields from an existing container. Provide the name of the item to clone.

edit

Change an existing entry.

happi edit [-h] name edits [edits ...]

Positional Arguments

name

Name of the item to edit

edits

Edits of the form: field=value

load

Open IPython terminal with a given device loaded.

happi load [-h] device_names [device_names ...]

Positional Arguments

device_names

The names of one or more devices to load

update

Update happi db with JSON payload.

happi update [-h] [json ...]

Positional Arguments

json

JSON payload.

container-registry

Print container registry.

happi container-registry [-h]

transfer

change the container of an item.

happi transfer [-h] name target

Positional Arguments

name

Name of the item to edit

target

Container to transfer item into
